Issues Found & Resolved

Fixed Today
ISSUE-001 BEFORE: NEOGEN STORE AFTER: NovaKeys Store All pages · PWA install
webmanifest shipped with NeoGen brand identity
site.webmanifest declared "name": "NEOGEN STORE", "short_name": "NEOGEN", and "theme_color": "#0099cc". Any user who installed the PWA to their home screen would see "NEOGEN" as the app name and a mismatched theme colour. Fixed to "name": "NovaKeys Store", "short_name": "NovaKeys", "theme_color": "#38BDF8", "background_color": "#F8FAFC".
neogen-theme-assets/icons/site.webmanifest
ISSUE-002 BEFORE: NG monogram AFTER: NK monogram Safari · macOS pinned tabs
Safari pinned-tab SVG displayed the NeoGen "NG" letterform
safari-pinned-tab.svg contained SVG paths drawing an N and a G — the NeoGen monogram. Replaced with correct NK letterforms. Any Safari user who pinned novakeys.store would have seen the competitor's initial in their tab bar.
neogen-theme-assets/icons/safari-pinned-tab.svg
ISSUE-003 BEFORE: NeoGen icon AFTER: NovaKeys NK icon 9 files · All icon sizes
Entire favicon suite was the NeoGen branded icon
All PNG icon assets — icon-16.png, icon-32.png, icon-48.png, icon-192.png, icon-512.png, apple-touch-icon.png, and favicon.ico — were the original NeoGen graphic, unchanged from the template copy.
