# theme-bridge — learnings

Append-only. Newest entry on top. Each entry: a date, a few non-obvious facts the next session would benefit from. No restating what's in `map.md` — only the *why* and the gotchas.

## 2026-05-07 — first pass

- **`#ng-queue` is theatre.** `assets/chrome/neogen.js:29–45` shows a JavaScript random walk in [8, 22] running on a 12-second interval, presented as the live "orders in queue" number in the sysbar. The adjacent `#ng-clock` IS real (Asia/Riyadh `toLocaleString`), which makes the bar feel authoritative as a whole. The mismatch is consumer-deceptive under KSA Consumer Protection rules and trips the trust posture the rest of the chrome works hard to establish. **Don't replicate this pattern; flag it for removal next time the chrome is touched.**

- **Postmeta keys are frozen at `_ng_*` even though symbols renamed to `nk_*`.** Phase-4 (commit `a6bf4ac chore(phase-4): retire ng_* compat shims`) retired the public-symbol `ng_*` shims, but `_ng_ar_title`, `_ng_gift_card_region`, `_ng_gift_card_brand` are *still in use* and must stay. They're written by external bulk-import scripts (`scripts/neogen-gift-cards-bulk.php`, `-brand-cats.php` per the comment at `theme-bridge.php:585`) and read by content-product.php, content-single-product.php, the brand-grid SQL (`theme-bridge.php:573–581`), and the gift-cards `pre_get_posts` filter (`theme-bridge.php:475–486`). Renaming the meta keys breaks live data. **Why:** the rename plan documented at `theme-bridge.php:13–16` explicitly excludes postmeta — symbol drift, not meta drift.

- **Chrome CSS file lives inside the plugin, not a mu-plugin.** Commit `05e07b3 feat(chrome): rebuild neogen.css/js inside the plugin, drop legacy mu-plugins refs` moved the asset out of `mu-plugins/`. Enqueue uses `NK_COMMERCE_URL` (plugin URL) — *not* `NG_THEME_ASSET_URL` (which still points at `<plugin>/includes/theme/neogen-theme-assets/`, kept for templates + icons + img). If you see a stale CSS reference at a `mu-plugins` path, it's pre-`05e07b3` code. Both URL constants are live and serve different bundles — don't unify them without checking what each path serves.

- **`.ng-rack-unit` had no CSS until 2026-05-07.** `nk_shop_category_tiles()` at `theme-bridge.php:336–433` emits rich nested category-card markup (id kicker, LED cluster, icon, title, count, browse link) but `neogen.css` only had rules for `.ng-rack` (small chip variant, lines 639–649) and `.ng-rack-led` / `.ng-rack-id` siblings. There was even a tell-tale comment at `neogen.css:660` — `/* Gift cards page (basic skeleton — full grid styled separately) ---- */` — meaning the dev knew the rule set was incomplete. Fixed in commit `bf2fd80`. **If you see comment markers like "styled separately" / "TBD" in a CSS file, treat them as missing-CSS BLOCKERs, not optional polish.**

- **`brand_ar` is misbranded.** `theme-bridge.php:39` has `'brand_ar' => 'نيوجين ستور'` — that's *NeoGen Store* in Arabic, copied from the sibling NeoGen project. The site is NovaKeys. Anywhere the chrome rendering pulls `nk_cr()['brand_ar']`, customers see the wrong Arabic brand. **Why this matters:** the chrome's stated brand integrity (legal name, CR, regulatory IDs) is loud and verifiable elsewhere on the page; a wrong AR brand is the kind of detail a Saudi customer notices. The fix is mechanical — change one literal — but verify the Arabic brand decision with the user before changing (pre-Sept 2025 the site may have been NeoGen).

- **`template_include` filter is chained at three priorities and the order is load-bearing.** Front-page template at 99 (line 2014), virtual-page template at 1001 (line 1822). The active theme (Blocksy child or `themes/novakeys`) attaches its own filter at 999 — the comment at `theme-bridge.php:1879–1885` explains why 1001 was chosen. Don't lower the virtual-page priority below 1000 without auditing what the active theme is doing at 999 — comment specifically calls out an `is_home()` blanket-rewrite to `app-shell.php` that previously clobbered legal pages.

- **Rewrite-rule registration is gated by a per-version flag.** `theme-bridge.php:1763–1767` writes `novakeys_rewrites_flushed_<ver>` after one flush per `NOVAKEYS_THEME_VERSION` value. **Adding a new info-page slug to `nk_info_pages()` will 404 in production until you also bump `NOVAKEYS_THEME_VERSION`.** Easy to miss because the local dev environment usually has rewrites flushed for unrelated reasons.

- **Newsletter form's POST action is hardcoded to `ng_newsletter_subscribe`.** `admin_post_ng_newsletter_subscribe` and `_nopriv_*` are the registered hooks (`theme-bridge.php:2207–2208`); the footer form's `<input name="action" value="ng_newsletter_subscribe">` matches. Renaming the hook to `nk_newsletter_subscribe` requires touching both the form markup AND any cached page output AND any in-flight POSTs — there's no compat layer for this one. **Why:** the action is what `admin-post.php` dispatches on, not a callable name. Treat as a frozen string until the next chrome rewrite.

- **Schema.org JSON-LD defers to a separate engine when active.** `theme-bridge.php:1645–1741` emits a Store + WebSite + WebPage `@graph`, but the first guard (line 1649) is `if ( ng_seo_engine_enabled() ) return`. There's a separate mu-plugin `mu-plugins/neogen-seo-engine.php` (referenced in the comment) that emits a more complete `@graph` when its R5 cutover toggle is on. **If schema is missing on the live site, check both this file AND the SEO engine** — disabling one without the other yields zero schema.

- **CR data has nested parent-LLC + regulatory blocks that show up in JSON-LD identifiers.** `NK_CR['parent']` (NTS LLC, Wyoming) and `NK_CR['regulatory']` (ZATCA `3145127947`, Council of Saudi Chambers `1238532`) are factual and emitted as Schema.org `PropertyValue` identifiers (`theme-bridge.php:1669–1689`). Don't speculate-add fields here — comment at line 30–34 says "do NOT edit to add speculative data; VAT / physical address / extra contacts should arrive via filters."

- **Light-mode is enforced before first paint at `wp_head` priority 0.** `theme-bridge.php:1562–1568` writes `data-prefers-color-scheme="light"` and strips `ct-dark`/`dark`/`dark-mode` classes on `<html>`. This beats the parent Blocksy dark-mode init script. **Phase-2c gate:** `ng_blocksy_dark_mode_allowed()` (external function, not defined here) lets the operator opt-in via Tools → NovaKeys Blocksy Handoff. When ON, this script no-ops and Blocksy's color-scheme picker takes over. If you see an unexpected dark flash, this is the gate to check.

- **Two GTM option keys in cascade — `nk_gtm_container_id` preferred, `ng_gtm_container_id` fallback.** Default falls back to `'GTM-PRTBSHTW'` if both are empty (`theme-bridge.php:1604–1608`, mirrored at 1626–1630 for the noscript iframe). That's a real container ID. **Don't lift this code into a different project without removing the default** — anything you instrument under that container shows up in the original GTM workspace.
